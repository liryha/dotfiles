Pretty Diff - Guide

Processing the JSX format from Facebook's React

Introduction

React.js is a JavaScript template framework coming out of Facebook. Perhaps the most exciting feature of this technology is its data format for defining templates called JSX. Essentially, JSX allows writing HTML template code directly into a JavaScript function using XML syntax.

JSX Pragma

JSX requires a pragma state to appear at the top of the code. When Pretty Diff's jspretty library encounters the pragma statement before any uncommented code it will automatically interpret the document as JSX format without any setting or option. The pragma is comment like the following:

/** @jsx React.DOM*/

For jspretty to recognize the pragma the following conditions must apply.

  1. The comment must start with /**.
  2. The comment must end with */.
  3. The comment must contain the case-sensitive string @jsx React.DOM.
  4. The comment must occur before any uncommented code.
  5. The comment may contain additional asterisks and white space characters, but not any other characters.

Markup embedded within JavaScript

If in JSX mode the jspretty library will expect angle braces to represent XML tags and not JavaScript operators. If angle braces need to represent operators then they must be contained by curly braces. Once an angle brace is encountered it is interpreted as an XML start tag expecting an XML end tag or a lone XML singleton. The markup language token will be isolated and sent to the markupmin library for minification or the markup_beauty library for beautification.
